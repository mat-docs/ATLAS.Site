# ATLAS Viewer Release: v11.5.2.947-W28

This release adds support for Kafka authentication on the Stream Recorder, virtual parameter support for the Stream Recorder and USB Logger, and Session Browser filter usability improvements, while fixing Stream Recorder auto-record and lap-naming regressions, PCU Dash and Scatterplot session handling, and several SQL Race and Session Browser stability issues.

<!-- more -->

!!! info "Evaluation Build Notice"
    This release is provided as an evaluation build, offering early access to a set of completed fixes and features ahead of a full production release. Evaluation builds are intended for validation and early feedback, and while the included changes are functionally complete, there may still be known limitations or issues that are addressed before the final release. We encourage users to report any problems or feedback encountered during evaluation, as this helps ensure the highest quality for the full release.

## New Features

### Kafka Authentication and Settings for Stream Recorder

The Stream Recorder now exposes Kafka publishing and consuming configuration, including authentication and authorisation settings, allowing connections to secured Kafka deployments to be configured directly from ATLAS. This release enables future support for Authentication settings.

### Virtual Parameters in Stream Recorder and USB Logger Recorder

Virtual parameters are now processed by the Stream Recorder and the USB Logger Recorder. Previously, virtual channels could be missing from recorded or offloaded sessions even though the underlying BIOS channel data was present.

### Export Data – Session Identifier and Auto-Increment for SQL Race Database Export

Added a text box to set a session identifier when exporting to a SQL Race Database, so repeated exports to the same database no longer produce indistinguishable copies of a session differing only by session key. The currently selected session is used as the default, so existing export behaviour is unchanged.

### Automation API – ActivateDisplay Method

Added an `ActivateDisplay` method to `DisplayServiceClient` in the Automation API, allowing displays to be programmatically focused/activated by ID, mirroring the existing `ActivatePage` functionality.

### Session Browser – Clear Filters

The Session Browser now shows which filters are active even when filter groups are collapsed, and adds a quick way to clear all active filters at once.

## Bug Fixes

### PCU Display Loses Selected Raw File After Refresh During Recording

Fixed an issue where the PCU Dash display's manually selected raw file was cleared after using "Refresh raw files" during recording, requiring the file to be re-selected.

### Stream Recorder Jumpy and High Memory Use with RDS Data

Fixed an issue causing the Stream Recorder to become jumpy and consume excessive memory when receiving RDS data.

### Highest Rate Parameters Not Shown Across Associated vTAG Sessions (#55829)

Fixed an issue where ATLAS did not display the highest rate of parameters when multiple vTAG-associated session files were loaded, depending on the order the associated sessions were attached.

### Reprogramming Stops Live Ethernet Telemetry Recorder

Fixed a regression where reprogramming a unit during a live Ethernet Telemetry recording closed the live session and stopped the recorder instead of pausing through the reprogram cycle.

### Empty Displays After "End Time is Less Than Start Time" Exception

Fixed a regression where ATLAS could show no data for a loaded session, despite the recorder having data, after an occasional end-time/start-time validation exception.

### Sorting by Date/Time Crash on Invalid Session Timestamp (#6546)

Fixed a crash in the Session Browser when sorting by date/time while a session had an invalid or corrupted date/timestamp value.

### PCU Dash – Unclear Message When Adding Parameters

Updated the message shown when attempting to add parameters to the PCU Dash display to a clear, fixed message.

### Scatterplot – Redundant Redraws

Removed a duplicate graphics invalidation triggered on every Scatterplot data update, reducing unnecessary rendering and CPU usage.

### Scatterplot – Greyed-Out Parameters After Removing Primary Appended Session

Fixed an issue where removing the primary session from a set with appended sessions left Scatterplot parameters greyed out and showing no data until the display was manually refreshed.

### Splash Screen Sizing at Different Windows Display Scaling

Fixed an issue where the splash screen did not size consistently across different Windows display scaling settings.

### Stream Recorder Auto-Record and Lap Naming Regressions

Fixed a set of related issues in the Stream Recorder: the first lap could be named incorrectly instead of "Out Lap", auto-record could start recording when the session identifier pattern did not match, and recordings could get stuck live requiring a manual stop.

### Live Session Load Failure via File Telemetry with Server Listener On

Fixed an issue where a live session created via File Telemetry with the server listener enabled could fail to load in the Session Browser with an arithmetic overflow error; the session loaded correctly only once it became historical.

### SQL Race Session TimeZone Always Returns UTC (#7002)

Fixed an issue where the `TimeZone` property on `SessionSummary` always returned "UTC" instead of the time zone in force at the time of recording.
