# ATLAS Viewer Release: v11.5.2.215-W15

This release adds improved Stream Recorder continuity and recorder usability, while fixing a range of data integrity, workbook saving, telemetry display, navigation, and USB Logger recording issues to ensure more reliable recording and playback.

<!-- more -->

!!! info "Evaluation Build Notice"
    This release is provided as an evaluation build, offering early access to a set of completed fixes and features ahead of the full production release planned for May. Evaluation builds are intended for validation and early feedback, and while the included changes are functionally complete, there may still be known limitations or issues that are addressed before the final release. We encourage users to report any problems or feedback encountered during evaluation, as this helps ensure the highest quality for the full release.

## New Features

### Consumer Group ID Support in Stream Recorder

Added support for configuring a Kafka consumer group ID in ATLAS, allowing stream recorders to resume reading live data from the last disconnect instead of always starting at the latest offset; when a valid group ID (alphanumeric with dots, dashes, or underscores) is provided in the connection details it is reused, otherwise a new group is created automatically.

### Recorder Configuration – Compact SQLite File Path Display

Updated the SQLite recorder configuration to display long database file paths using a middle ellipsis (for example `C:\...\sessions_12.sqlite`), ensuring the filename and extension are always visible, with the full path shown on hover and restored when editing.

## Bug Fixes

### SQL Race API - Duplicate Timestamp Handling 

Fixed an issue where duplicate samples could trigger a “same key already added” exception and cause the parameter to appear blank at certain zoom levels, ensuring all samples are displayed correctly without errors.

### USB Logger Recorder – No Data Recorded

Fixed a regression where the USB Logger Recorder could produce sessions with no recorded data, ensuring all data is correctly captured and available when opening the session.

### CAN Messages With DLC < 8 Not Handled Correctly

Implemented better error handling when where SQL Race fails to write CAN data to disk with a DLC less than 8. 

### Linked Pages – Window Layout Reset (#4403, #5653)

Fixed an issue where opening a linked page via CTRL‑click would resize and stack all page windows, ensuring the target page opens without altering the existing window layout.

### Incorrect Boolean Value Display (#7054)

Fixed an issue where Boolean telemetry parameters could display incorrect values in ATLAS while the underlying logged data was correct.

### New Workbook Lost on Close (#7268)

Fixed an issue where workbooks were not saved when closing ATLAS even after selecting Yes at the save prompt, ensuring new workbooks are correctly saved without requiring an initial Save or Save As.

### USB Logger Recorder – File List and Progress State Restored During Recording

Fixed an issue where reopening the USB Logger Recorder during an active recording showed no files and lost selection; the file list, selection, and per‑file progress status are now correctly restored when the window is reopened, with progress visible throughout recording.

## Known Issues

### Scatterplot Z‑Axis Colour Bar Missing

In some cases, scatterplots with a Z‑axis colour bar saved on a non‑active page may reopen without the colour bar once data has loaded; switching page association, adding another session, or making a small edit to the scatterplot will refresh it and restore the colour bar.
